# Capítulo III: Requirements Specification

## User Stories

| **Epic ID** | **Título**                                                              | **Descripción**                                                                                                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **EP01**    | Gestión de identidad y onboarding personalizado                          | Como usuario de Viora (Productor o Especialista), quiero gestionar mi acceso y perfil de manera segura y diferenciada, para disponer de una experiencia de bienvenida adaptada a mis necesidades profesionales y activar los beneficios del periodo de prueba sin fricciones.           |
| **EP02**    | Configuración de activos: gestión de parcelas y visibilidad profesional | Como usuario de Viora (Productor o Especialista), quiero configurar y gestionar mis activos principales (delimitación de parcelas agrícolas o publicación de mi portafolio profesional), para habilitar el monitoreo satelital, alimentar el motor predictivo con el historial agronómico necesario, y activar mi visibilidad en el ecosistema de conexión profesional.                        |
| **EP03**    | Dashboard integrado de monitoreo agrometeorológico y predicción         | Como Productor Olivarero, quiero visualizar un panel de control interactivo (dashboard) que consolide la información climática, el estado de salud satelital de mi cultivo y las proyecciones de cosecha, para evaluar la situación real de mi parcela y tomar decisiones preventivas frente a anomalías ambientales. |
| **EP05**    | Vigilancia fitosanitaria y radar de prospección técnica                 | Como usuario de Viora, quiero contar con un sistema de reporte, validación y visualización de amenazas biológicas, para gestionar alertas tempranas en la comunidad y permitir a los técnicos identificar oportunidades de intervención en zonas de riesgo.                                                           |
| **EP06**    | Matchmaking y solicitud de intervención fitosanitaria                   | Como usuario de Viora, quiero buscar y solicitar ayuda técnica (Productor) o gestionar las solicitudes entrantes (Especialista), para establecer un primer contacto ante una emergencia en el cultivo.                                                                                                                |
| **EP07**    | Evaluación técnica, cotización y revelación de contacto                 | Como usuario de Viora, quiero formalizar la prestación del servicio mediante la revisión técnica del fundo, el envío de una propuesta económica y la visualización de los datos de contacto directo, para iniciar la mitigación de la plaga.                                                                         |
| **EP08**    | Moderación automatizada y sistema de strikes                            | Como usuario de Viora, quiero contar con un sistema de reporte de inconductas que evalúe automáticamente el comportamiento en la plataforma, para que los usuarios conflictivos sean sancionados sistemáticamente mediante la acumulación de reportes.                                                    |
| **EP09**    | Diagnóstico fitosanitario en campo y prescripción técnica               | Como Especialista en Control de Plagas, quiero revisar el historial de la parcela y registrar los hallazgos de mi inspección física, para poder emitir una prescripción precisa en el momento climático óptimo.                                                                                                       |
| **EP10**    | Ejecución de la intervención y trazabilidad operativa                   | Como usuario de Viora, quiero registrar las acciones físicas realizadas en la parcela y los costos asociados, para mantener actualizada la trazabilidad del fundo y establecer un plan de manejo a futuro.                                                                                 |
| **EP11**    | Cierre del servicio, retroalimentación y actualización de portafolio    | Como usuario de Viora, quiero evaluar el impacto del servicio recibido indicando mi disposición a recontratar al técnico, para cerrar formalmente la asesoría y alimentar el algoritmo de recomendación del marketplace.                                                                                        |
| **EP12**    | Gestión de perfil, ajustes de cuenta y programa de afiliados            | Como usuario autenticado, quiero editar la información de mi perfil, gestionar mi disponibilidad y participar en el programa de referidos, para mantener mis datos actualizados y obtener beneficios por recomendar la plataforma.                                                                                    |
| **EP13**    | Gestión de suscripciones, pagos y facturación                           | Como usuario de Viora, quiero gestionar mi plan de suscripción y métodos de pago de forma segura, para mantener mi acceso a la plataforma o solicitar reembolsos según las políticas de garantía.                                                                                                                     |
| **EP14**    | Descubrimiento y propuesta de valor (hero, problema y solución)         | Como visitante, quiero entender de inmediato qué es Viora, cómo resuelve la incertidumbre climática y el enfoque reactivo en la agricultura, para decidir si la plataforma me es útil y navegar hacia la aplicación principal.                                                                                        |
| **EP15**    | Funcionalidades segmentadas, trial ilimitado y afiliados                | Como visitante, quiero ver qué herramientas me ofrece la plataforma según mi rol y conocer los planes, para entender el acceso total del periodo de prueba y los beneficios por recomendarla.                                                                                              |
| **EP16**    | Prueba social, respaldo institucional y navegación (footer)             | Como visitante cauteloso, quiero leer testimonios de otros usuarios, conocer al equipo desarrollador y tener enlaces de contacto, para sentir la confianza técnica necesaria antes de registrar mis datos profesionales o los de mi fundo.                                                                            |
| **EP17**    | API de identidad, autenticación y perfiles (iam)                        | Como Developer, quiero implementar un sistema de Identity and Access Management (IAM) basado en tokens de seguridad, para garantizar que los Productores y Especialistas se registren de forma segura, gestionen sus perfiles y tengan acceso restringido según su rol.                                                |
| **EP18**    | API del core agronómico y procesamiento externo                         | Como Developer, quiero desarrollar el módulo central agronómico integrado a servicios externos, para gestionar el registro de parcelas, automatizar la ingesta de datos climáticos/satelitales y alimentar los algoritmos predictivos.                                                                                |
| **EP19**    | API del marketplace y flujo transaccional                               | Como Developer, quiero exponer los endpoints del motor transaccional del marketplace, para habilitar el algoritmo de búsqueda de expertos, el intercambio de cotizaciones y el registro de la bitácora de intervenciones.                                                                                             |
| **EP20**    | API de eventos comunitarios y moderación autónoma                       | Como Developer, quiero construir el motor de difusión de alertas fitosanitarias y el sistema de strikes, para enviar notificaciones de riesgo a los usuarios y castigar inconductas automáticamente sin intervención humana.
